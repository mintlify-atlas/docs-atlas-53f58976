---
title: "Get started with MechCommander 2 Engine"
description: "Learn how to clone, build, and run the MechCommander 2 open-source engine on Linux or Windows in four steps using CMake, SDL2, and OpenGL."
---

This quickstart walks you through everything needed to get the MechCommander 2 open-source engine running — from installing dependencies to launching the game. You will need a copy of the original MechCommander 2 game data files, which are not included in this repository.

<Note>
  The original MC2 game data is required to run the engine. See the [data assets guide](/building/data-assets) for instructions on preparing game data using the mc2srcdata repository.
</Note>

<Steps>
  <Step title="Install prerequisites">
    The engine requires CMake 3.10+, a C++11-capable compiler, and several runtime libraries.

    <Tabs>
      <Tab title="Debian / Ubuntu">
        ```bash
        sudo apt-get install \
          cmake build-essential \
          libsdl2-dev libsdl2-mixer-dev libsdl2-ttf-dev \
          libglew-dev libgl1-mesa-dev \
          zlib1g-dev
        ```
      </Tab>
      <Tab title="Fedora / RHEL">
        ```bash
        sudo dnf install \
          cmake gcc-c++ \
          SDL2-devel SDL2_mixer-devel SDL2_ttf-devel \
          glew-devel mesa-libGL-devel \
          zlib-devel
        ```
      </Tab>
      <Tab title="Windows">
        Install **Visual Studio 2022** with C++ workload, and **CMake**.

        Download pre-built third-party libraries or extract the included `3rdparty.zip` from the repository root. See the [Windows build guide](/building/windows) for the full dependency setup.
      </Tab>
    </Tabs>
  </Step>

  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/alariq/mc2.git
    cd mc2
    ```
  </Step>

  <Step title="Configure and build">
    <Tabs>
      <Tab title="Linux">
        ```bash
        mkdir build64 && cd build64
        cmake -DCMAKE_PREFIX_PATH=/path/to/dependencies ..
        make -j$(nproc)
        ```

        For a debug build with extra checks:
        ```bash
        cmake -DCMAKE_BUILD_TYPE=Debug -DCMAKE_PREFIX_PATH=/path/to/dependencies ..
        make -j$(nproc)
        ```

        After the main build, build the string resources library:
        ```bash
        cd ../res
        mkdir build64 && cd build64
        cmake ..
        make
        ```
      </Tab>
      <Tab title="Windows (Visual Studio 2022)">
        Open the **x64 Native Tools Command Prompt for VS 2022**, then:

        ```powershell
        md build64
        cd build64
        cmake.exe -G "Visual Studio 17 2022" -DCMAKE_PREFIX_PATH=C:\path\to\3rdparty -DCMAKE_LIBRARY_ARCHITECTURE=x64 ..
        ```

        Open the generated `.sln` in Visual Studio and build the solution. Then build the string resources DLL:

        ```powershell
        cd ..\res
        md build64
        cd build64
        cmake.exe -G "Visual Studio 17 2022" -DCMAKE_LIBRARY_ARCHITECTURE=x64 ..
        ```

        Build and copy `mc2res.dll` next to `mc2.exe`.
      </Tab>
    </Tabs>

    For complete platform-specific instructions, see the full build guides:

    <CardGroup cols={2}>
      <Card title="Linux build guide" icon="linux" href="/building/linux">
        Full step-by-step Linux build with all CMake options.
      </Card>
      <Card title="Windows build guide" icon="windows" href="/building/windows">
        Complete Windows setup including third-party library preparation.
      </Card>
    </CardGroup>
  </Step>

  <Step title="Get game data and run">
    Clone the data source repository and build the game assets:

    ```bash
    git clone https://github.com/alariq/mc2srcdata.git
    cd mc2srcdata/build_scripts
    # Copy the built data tools here first (aseconv, makefst, makersp, pak, text_tool)
    make all
    ```

    Copy the resulting `assets/`, `data/`, and configuration files to your executable directory. Then run the engine:

    <Tabs>
      <Tab title="Linux">
        ```bash
        ./build64/mc2
        ```
      </Tab>
      <Tab title="Windows">
        ```powershell
        # From your executable directory with mc2.exe, mc2res.dll, and all game data
        mc2.exe
        ```
      </Tab>
    </Tabs>

    <Note>
      See the [data assets guide](/building/data-assets) for a full walkthrough of the asset build pipeline, including the correct directory structure the engine expects.
    </Note>
  </Step>
</Steps>
